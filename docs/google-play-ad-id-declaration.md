# Google Play Console 広告ID申告ガイド

## 📋 現在の状況

**質問**: 「アプリで広告IDを使用していますか？」

**回答**: **いいえ** ✅

## ✅ なぜ「いいえ」で問題ないのか

1. **現在のアプリは広告IDを使用していない**
   - AndroidManifest.xmlに`AD_ID`権限が宣言されていない
   - 広告関連のSDK（Google Mobile Ads SDK等）が含まれていない
   - 広告IDを取得するコードがない

2. **将来の変更は可能**
   - いつでも「はい」に変更できる
   - 広告を追加する際に申告を更新すればよい

## 🔄 将来広告を追加する場合

### Step 1: 実装完了後に申告を変更

1. Google Play Consoleにログイン
2. **アプリの内容** → **データの収集とセキュリティ**
3. **広告 ID**セクションで「はい」に変更
4. 理由: 「無料ユーザー向けにGoogle AdMobを使用して広告を表示するため」

### Step 2: AndroidManifest.xmlに権限追加

```xml
<uses-permission android:name="com.google.android.gms.permission.AD_ID"/>
```

### Step 3: リリース前に確認

- マニフェストに`AD_ID`権限が含まれていることを確認
- 広告SDKが正常に動作することを確認
- テスト広告が表示されることを確認

## ⚠️ 重要なポイント

1. **実装前に申告を変更しない**
   - 広告実装が完了してから申告を変更する
   - 実装前に「はい」にすると、マニフェストに権限がない場合にリリースがブロックされる

2. **現在は「いいえ」で問題なし**
   - 将来広告を追加する予定があっても、現在は「いいえ」でOK
   - 実装時に変更すればよい

3. **申告の変更はいつでも可能**
   - アプリの更新時に変更できる
   - 新しいリリースで申告を更新すればよい

## 📝 回答例

### 現在（広告なし）
```
アプリで広告IDを使用していますか？
→ いいえ

理由: 現在、アプリは広告IDを使用していません。
広告関連のSDKは含まれておらず、AndroidManifest.xmlにも
AD_ID権限が宣言されていません。
```

### 将来（広告追加後）
```
アプリで広告IDを使用していますか？
→ はい

理由: 無料ユーザー向けにGoogle AdMobを使用して
広告を表示するため。AndroidManifest.xmlに
com.google.android.gms.permission.AD_ID権限を宣言しています。
```

## ✅ 結論

**現在は「いいえ」と回答して問題ありません。**

将来的に広告を追加する際は、実装完了後に申告を「はい」に変更し、AndroidManifest.xmlに権限を追加すればOKです。

申告の変更はいつでも可能なので、焦る必要はありません。
